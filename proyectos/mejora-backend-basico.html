Desarrollo Backend

Sobre el proyecto

En el sitio web de DUDUDU notamos que la arquitectura era poco escalable, así que se trabajó en el backend.

Mis funciones: Desarrollar el sitio web empezando por el backend y convertir los archivos html a pug.

El Frontend se mantuvo casi igual, el backend ayudó en partes al desarrollo y sin duda mejoraría la escalabilidad del sitio web. Digo mejoraría porque no se pudo implementar, porque el hosting no dejaba trabajar con node.js en el plan en que se estaba.


Herramientas utilizadas

Node.js

Usé Node.js para conectarse con el servidor, porque creo que actualmente es mejor que php, y como lenguaje de programación no se necesita cambiar, se sigue utilizando JavaScript. Dentro de Node.js usé Express.js para que aligere el desarrollo. Node.js junto con MySql son los que hacen que sea escalable el sitio web.


Pug

Al instalar el motor de platillas se configuro automáticamente a archivos Jade (Que se rige por la misma documentación que Pug y la documentación de Jade quedó discontinuado), ademas es un lenguaje de programación popular actualmente. Esta es la parte a la que me refería que "el backend ayudó en partes al desarrollo", lo hizo más simplificado.


MySQL

Usamos MySQL mediante phpMyAdmin para tener ciertos datos como títulos, párrafos que se repiten en dos o más páginas, en una base de datos, para poder usarlos conectándose mediante Node.js.


FileZilla

Al ultimo se usó FileZilla para estandarizar ciertos parámetros.
